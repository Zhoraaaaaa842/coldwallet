# ColdVault ETH — Холодный кошелёк Ethereum

Программный cold wallet с air-gapped подписью транзакций через USB.

---

## Архитектура

```
┌─────────────────┐     USB      ┌──────────────────┐
│  Онлайн-ПК      │◄────────────►│  Офлайн-ПК       │
│  (Десктоп-App)   │   pending/   │  (sign_offline)   │
│                  │   signed/    │                   │
│  • Баланс       │              │  • Приватный ключ │
│  • Создание TX  │              │  • Подпись TX     │
│  • Broadcast    │              │  • Мнемоника      │
└─────────────────┘              └──────────────────┘
```

Приватный ключ **никогда не покидает** офлайн-среду.

## Установка

### 1. Зависимости

```bash
pip install -r requirements.txt
```

### 2. Создание кошелька на USB

> Запускайте на **офлайн-ПК** (без интернета) для максимальной безопасности.

```bash
python installer/install_to_usb.py
```

Скрипт:
- Обнаружит все подключённые USB-накопители
- Предложит выбрать нужный
- Создаст структуру ColdVault
- Сгенерирует (или импортирует) кошелёк
- Зашифрует ключ паролем и сохранит на USB

#### Как установить кошелёк на вторую (и каждую следующую) флешку

Каждая флешка — **отдельный независимый кошелёк** (разные ключи, разные адреса).
Чтобы создать кошелёк на новой флешке:

1. Вставьте новую флешку в ПК
2. Запустите установщик ещё раз:
   ```bash
   python installer/install_to_usb.py
   ```
3. Скрипт покажет список всех USB — выберите нужную
4. Введите **новый пароль** для этого кошелька
5. Запишите мнемонику (24 слова) — она отображается **только один раз**

> ⚠️ Если хотите один и тот же кошелёк на двух флешках (резервная копия) — при установке выберите «Импортировать существующий» и введите мнемонику от первой флешки.

### 3. Запуск десктоп-приложения (на онлайн-ПК)

```bash
python run_desktop.py
```

При первом запуске:
1. Подключите USB с кошельком
2. Нажмите **«Обнаружить USB»** — приложение найдёт `wallet.vault`
3. Введите пароль — кошелёк разблокируется
4. Адрес и баланс появятся на главном экране

### 4. Сборка EXE (опционально)

```bash
python build_exe.py
```

Результат: `dist/ColdVault.exe`

## Рабочий процесс

### Отправка ETH (полный цикл)

```
[Онлайн-ПК]  →  [USB]  →  [Офлайн-ПК]  →  [USB]  →  [Онлайн-ПК]
  Создать TX     pending/    Подписать       signed/    Broadcast
```

**Подробно:**

1. **Онлайн-ПК** → вкладка **«Отправить»**
   - Введите адрес, сумму, gas
   - Нажмите **«Создать TX → USB»** — файл сохранится в `pending/`
2. Безопасно извлеките USB, вставьте в офлайн-ПК
3. **Офлайн-ПК** → вкладка **«Подписать & Отправить»**
   - Нажмите **«Сканировать pending/ на USB»**
   - Нажмите **«✎ Подписать»** рядом с файлом → введите пароль → подтвердите
   - TX подписана и сохранена в `signed/`
4. Вставьте USB обратно в онлайн-ПК
5. **Онлайн-ПК** → вкладка **«Подписать & Отправить»**
   - Нажмите **«Сканировать signed/ на USB»**
   - Нажмите **«▶ Отправить в сеть»**

### Получение ETH (QR-код)

1. Вкладка **«Получить (QR)»** → нажмите **«Сгенерировать QR»**
2. Покажите QR отправителю или сохраните в PNG
3. Чтобы заполнить форму отправки из чужого QR:
   - Загрузите изображение QR или введите URI вручную
   - Нажмите **«Заполнить форму Отправки»** — адрес и сумма подставятся автоматически

## Структура USB

```
USB/
└── ColdVault/
    ├── wallet.vault        # Зашифрованный кошелёк (AES-256-GCM)
    ├── config.json         # Конфигурация (chain_id, сеть)
    ├── pending/            # Неподписанные TX (создаются на онлайн-ПК)
    │   └── tx_0_17759429.json
    └── signed/             # Подписанные TX (создаются на офлайн-ПК)
        └── tx_0_17759429_signed.json
```

## Несколько флешек — таблица сценариев

| Сценарий | Действие |
|---|---|
| Новый независимый кошелёк | `install_to_usb.py` → выбрать флешку → новый пароль |
| Резервная копия (тот же адрес) | `install_to_usb.py` → «Импортировать» → ввести мнемонику |
| Разные адреса для разных целей | Установить на каждую флешку отдельно с разными паролями |
| Проверить, что на флешке уже есть кошелёк | Открыть `USB/ColdVault/wallet.vault` — файл существует |

## Безопасность

| Компонент | Технология |
|-----------|-----------|
| Шифрование ключа | AES-256-GCM |
| Деривация пароля | PBKDF2-SHA256 (600 000 итераций) |
| Мнемоника | BIP-39 (24 слова, 256 бит) |
| HD-деривация | BIP-44 (m/44'/60'/0'/0/0) |
| Транзакции | EIP-1559 + Legacy, ECDSA secp256k1 |
| QR-формат | EIP-681 (ethereum:0x...@chainId?value=) |

Полный аудит: [SECURITY_AUDIT.md](SECURITY_AUDIT.md)

## Предупреждение

Это **программный** холодный кошелёк. Он не заменяет аппаратные кошельки (Ledger, Trezor) с чипом Secure Element. Используйте для обучения и умеренных сумм. Для крупных сумм используйте аппаратный кошелёк.
